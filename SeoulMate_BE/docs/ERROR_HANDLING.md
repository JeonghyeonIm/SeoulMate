# ERROR_HANDLING

## 문서 목적

프로젝트에서 발생하는 에러 상황별 HTTP 상태코드와 응답 메시지를 정리한다.

## 공통 에러

### 공통 인증 에러

아래 인증 에러는 `recommendation`, `place`, `user` 라우터의 보호된 엔드포인트 전반에 공통 적용된다.

| 상태코드 | 메시지                            | 발생 조건                                                                     |
| -------- | --------------------------------- | ----------------------------------------------------------------------------- |
| 401      | 인증 토큰이 필요합니다.           | `Authorization: Bearer <token>` 헤더가 없거나 형식이 맞지 않을 때             |
| 401      | 인증된 사용자를 찾을 수 없습니다. | access token은 유효하지만 토큰의 사용자 ID에 해당하는 사용자가 없을 때        |
| 401      | 유효하지 않은 토큰입니다.         | access token 검증 실패, 만료, 위변조 등 인증 미들웨어에서 예외가 날 때        |
| 401      | 로그인이 필요합니다.              | 인증 미들웨어를 통과하지 못했거나, 컨트롤러에서 `req.user`가 없다고 판단할 때 |

### 공통 서버 에러

| 상태코드 | 메시지                    | 발생 조건                                   |
| -------- | ------------------------- | ------------------------------------------- |
| 500      | 서버 오류가 발생했습니다. | 명시적으로 처리되지 않은 예외가 발생했을 때 |

## Auth API

### POST `/api/auth/signup`

| 상태코드 | 메시지                                                  | 발생 조건                                                                   |
| -------- | ------------------------------------------------------- | --------------------------------------------------------------------------- |
| 400      | 요청 본문은 객체여야 합니다.                            | 요청 body가 객체가 아닐 때                                                  |
| 400      | 유효한 email 형식이 아닙니다.                           | `email`이 문자열이 아니거나 이메일 형식이 아닐 때                           |
| 400      | nickname은 2자 이상 10자 이하여야 합니다.               | `nickname` 길이가 조건을 만족하지 않을 때                                   |
| 400      | password는 8자 이상이어야 합니다.                       | `password` 길이가 8자 미만일 때                                             |
| 400      | preferences.vibes는 null 또는 문자열 배열이어야 합니다. | `preferences` 구조가 잘못됐거나 `vibes`가 `null` 또는 문자열 배열이 아닐 때 |
| 400      | 허용되지 않은 vibes 값이 포함되어 있습니다.             | `preferences.vibes`에 허용 목록 외 값이 있을 때                             |
| 409      | 이미 사용 중인 이메일입니다.                            | 같은 이메일의 로컬/기존 계정이 이미 존재할 때                               |
| 409      | 이미 사용 중인 닉네임입니다.                            | 같은 닉네임이 이미 존재할 때                                                |
| 500      | 서버 오류가 발생했습니다.                               | 회원 생성, 비밀번호 해시, DB 처리 중 미처리 예외가 날 때                    |

| 500      | 서버 오류가 발생했습니다.                 | 인증 처리 중 미처리 예외가 날 때                                   |

### POST `/api/auth/refresh`

| 상태코드 | 메시지                             | 발생 조건                                                                  |
| -------- | ---------------------------------- | -------------------------------------------------------------------------- |
| 401      | refresh token 쿠키가 필요합니다.   | refresh token 쿠키가 없거나 빈 값일 때                                     |
| 401      | 이미 로그아웃된 토큰입니다.        | refresh token이 블랙리스트에 등록되어 있을 때                              |
| 401      | 존재하지 않는 사용자입니다.        | refresh token의 사용자 ID에 해당하는 사용자가 없을 때                      |
| 401      | 유효하지 않은 refresh token입니다. | refresh token 검증 실패, 만료, 위변조 등 refresh 처리 중 일반 예외가 날 때 |
| 500      | 서버 오류가 발생했습니다.          | 블랙리스트 조회, 사용자 조회 등에서 미처리 예외가 날 때                    |

## Recommendation API

### POST `/api/recommendations/recommend`

| 상태코드 | 메시지                                                                                                                    | 발생 조건                                                                |
| -------- | ------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------ |
| 400      | budget 값은 정수여야 합니다.                                                                                              | `budget`이 정수가 아닐 때                                                |
| 400      | budget 값은 0부터 200000까지 5000원 단위이거나 200001이어야 합니다.                                                       | `budget` 범위 또는 단위가 잘못됐을 때                                    |
| 400      | 유효한 dateTime 형식이 아닙니다.                                                                                          | `dateTime`이 문자열이 아니거나 파싱 불가능할 때                          |
| 400      | 날짜는 현재 시각 이후 10일 이내만 입력 가능합니다.                                                                        | `dateTime`이 과거이거나 10일 초과 미래일 때                              |
| 400      | vibes 값은 필수입니다.                                                                                                    | 구조화 요청에서 `vibes`가 아예 없을 때                                   |
| 400      | vibes 값은 문자열 배열이어야 합니다.                                                                                      | `vibes`가 문자열 배열이 아닐 때                                          |
| 400      | region 값은 필수입니다.                                                                                                   | 구조화 요청에서 `region`이 비어 있을 때                                  |
| 400      | duration 값은 lte-2h, gt-2h-lte-4h, gt-4h-lte-6h, gt-6h-lte-8h, gt-8h-lte-10h, gt-10h-lte-12h, gt-12h 중 하나여야 합니다. | `duration`이 허용 값이 아닐 때                                           |
| 503      | 현재 장소 검색 서비스를 이용할 수 없습니다. 잠시 후 다시 시도해주세요.                                                    | 추천 그래프의 Kakao 장소 검증 단계에서 Kakao API 쿼터 초과가 발생했을 때 |
| 500      | 서버 오류가 발생했습니다.                                                                                                 | 추천 그래프, 코스 저장, DB 처리 등에서 미처리 예외가 날 때               |
